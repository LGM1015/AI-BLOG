# 提示词工程入门：从基础到实战

## 什么是提示词工程？

提示词工程（Prompt Engineering）是与大语言模型有效沟通的艺术和科学。通过精心设计的提示词，我们可以引导 AI 生成更准确、更有用的输出。

## 基础原则

### 1. 清晰明确

❌ **模糊提示**：
```
写一些关于编程的内容
```

✅ **明确提示**：
```
用 Python 写一个函数，计算斐波那契数列的第 n 项，返回结果并附带时间复杂度分析
```

### 2. 结构化输出

指定输出的格式和结构：

```
请列出 5 个提高代码可读性的方法，以 JSON 格式返回：
{
  "methods": [
    {"name": "方法名", "description": "描述", "example": "示例代码"}
  ]
}
```

### 3. 上下文丰富

提供足够的背景信息：

```
我正在开发一个电商网站，使用 React + Node.js 技术栈。
请推荐 3 个适合处理高并发订单的支付接口，并说明集成注意事项。
```

## 进阶技巧

### 思维链提示 (Chain of Thought)

让模型展示推理过程：

```
问题：一个商店有 10 个苹果，卖了 3 个，又进货 5 个，现在有多少苹果？
请逐步推理，最后给出答案。
```

### 少样本提示 (Few-Shot)

提供示例让模型学习模式：

```
将以下句子改写成更正式的表达：
例子1：
输入：这东西挺好的
输出：此产品具有较高的实用价值

例子2：
输入：太贵了
输出：价格方面可能需要进一步考量

现在请将以下句子改写：
输入：这个方案不太行
输出：
```

### 角色扮演

让模型扮演特定角色：

```
你是一位资深全栈工程师，有 10 年开发经验。
请评审以下代码，指出潜在问题并提供优化建议：

[代码片段]
```

## 实战模板

### 1. 代码审查模板

```
你是一位代码审查专家。请审查以下代码，关注：
1. 代码安全性
2. 性能问题
3. 代码可读性
4. 最佳实践

语言：[编程语言]
代码：
[粘贴代码]
```

### 2. 学习助手模板

```
你是一位耐心的技术导师。请用通俗易懂的方式解释：
[概念名称]

要求：
- 从基础讲起
- 使用生活中的类比
- 提供实际例子
- 最后留一个思考题
```

### 3. 写作助手模板

```
你是一位专业的技术文档工程师。请帮我润色以下技术文档：

标题：[文档标题]
目标读者：[目标群体]
语气：[专业/友好/简洁]

内容：
[文档内容]

请检查：
1. 语法和拼写
2. 表达清晰度
3. 技术准确性
4. 整体结构
```

## 常见陷阱

1. **过度依赖**：不要所有问题都问 AI，要保持独立思考
2. **忽略验证**：AI 可能产生幻觉，重要信息要核实
3. **隐私泄露**：不要向 AI 提供敏感信息
4. **盲目采纳**：批判性评估 AI 的建议

## 练习建议

1. 从简单任务开始，逐步尝试复杂场景
2. 对比不同提示词的效果差异
3. 建立自己的提示词模板库
4. 关注 AI 社区的最新技巧分享
