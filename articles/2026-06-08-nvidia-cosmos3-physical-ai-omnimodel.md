2026年6月，台北国际电脑展（Computex）期间，NVIDIA CEO 黄仁勋在GTC Taipei主题演讲中正式发布**NVIDIA Cosmos 3**，这是全球首个**全模态物理AI开源 omnimodel**——一个能同时理解和生成文本、图像、视频、音频，并**直接输出机器人动作信号**的统一模型。消息发布当天，NVIDIA股价应声上涨，机器人概念股集体拉升，具身智能赛道再次成为全球AI产业最受关注的焦点。

这不仅是一款新模型的发布，更可能是物理AI（Physical AI）训练方式的一次结构性转折。

## 一、什么是 Omnimodel？为什么它和以前的模型都不一样

在说 Cosmos 3 之前，有必要先理解它所提出的 **omnimodel**（全模态模型）概念究竟意味着什么。

传统AI模型是"单模态"或"有限多模态"的：文本模型只处理文字，图像模型只处理像素，语音模型只处理声波。2023-2025年间的多模态模型——如GPT-4V、Gemini Pro vision——实现了文本和图像的联合理解，但依然存在明显局限：**输入和输出始终局限在数字世界**，模型能"看懂"机器人传回的画面，但无法直接告诉机器人"怎么动"。

Cosmos 3 打破了这道壁垒。它的 omnimodel 架构将输入/输出模态扩展到了**动作（Action）** 这一物理世界最关键的信号维度：

| 模型类型 | 典型代表 | 能理解的输入 | 能生成的输出 |
|---------|---------|------------|------------|
| 纯语言模型 | GPT-4、Claude | 文本 | 文本 |
| 多模态模型 | GPT-4V、Gemini | 文本+图像+视频 | 文本+图像 |
| **Omnimodel** | **Cosmos 3** | **文本+图像+视频+音频+动作** | **文本+图像+视频+音频+动作** |

这是业界首次有模型将**物理动作**纳入原生输入/输出模态。对机器人而言，这意味着模型不再只是"旁观者"，而是真正能够参与物理世界交互的"参与者"。

## 二、技术架构：双塔 Mixture-of-Transformers

Cosmos 3 采用了 NVIDIA 称之为**双塔 Mixture-of-Transformers（MoT）** 的架构设计，这是其能够实现全模态统一处理的技术基础。

简单来说，MoT 架构的核心思想是**稀疏激活**——不是让所有参数都参与每次推理，而是根据输入模态动态选择最相关的"专家"子网络进行处理。这带来的直接好处是：

- **训练效率大幅提升**：稀疏激活意味着计算资源的按需分配，而不是每次都全量消耗
- **推理速度两档分化**：提供"Nano"（极速推理，毫秒级响应）和"Super"（高精度推理）两个版本，前者适合实时机器人控制，后者适合离线仿真训练

NVIDIA 官方表示，在 Cosmos 3 的支持下，物理AI系统的**训练和评估周期可以从数月缩短到数天**，这对于需要反复迭代的机器人团队来说是颠覆性的效率提升。

## 三、五大运行模式：一模型多用途

Cosmos 3 并不是一个单一模型，而是一个**模型族**，通过不同的输入输出配置可以实现五种核心运行模式：

### 1. 视觉-语言模型（Vision-Language Model）

Cosmos 3 可以像 GPT-4V 一样理解图像和文本输入，完成视觉推理、问答、场景描述等任务。这意味着在机器人系统中，它可以直接理解摄像头捕获的画面，并给出语义层面的理解和判断。

### 2. 文生图 / 图生视频（Text-to-Image / Image-to-Video）

Cosmos 3 支持文本到图像生成，以及图像到视频动画生成。NVIDIA 将其定位为**合成数据生成引擎**——机器人训练需要大量真实场景数据，但采集成本极高，Cosmos 3 可以根据文字描述生成多样化的高质量训练视频，大幅降低数据成本。

### 3. 视频预测（Video-to-Video / Future Prediction）

给定一段当前视频，Cosmos 3 可以预测未来数秒的物理世界演变。这是机器人路径规划和自动驾驶场景预测的核心能力。例如，输入当前道路视频，模型预测行人和车辆下一秒的移动轨迹，从而提前规划避让路径。

### 4. 策略/动作模型（Policy / World-Action Model）

这是 Cosmos 3 最具革命性的模式——**直接输出动作信号**。给定当前环境状态（图像、视频），模型输出机器人下一步应该执行的具体动作（如"机械臂向左旋转15度"）。这是真正意义上将感知与执行连接在一起的端到端模型。

### 5. 前向/逆向动力学模型（Forward/Inverse Dynamics Model）

用于机器人仿真环境构建和动作模仿学习，支持在虚拟环境中快速验证控制策略的有效性，再迁移到真实机器人上。

## 四、为什么开源？NVIDIA 的生态战略

值得特别关注的是，NVIDIA 选择以 **OpenMDW-1.1** 许可证开源 Cosmos 3，权重已在 Hugging Face 上线，且可以在 build.nvidia.com 上直接在线体验（无需GPU）。

这背后是 NVIDIA 一套清晰的生态战略：

**第一，抢占物理AI的标准基础设施。** 机器人创业公司不需要从零训练世界模型，直接基于 Cosmos 3 微调专属于自己的控制策略。这意味着 NVIDIA 在 AI 芯片之外，正在成为物理AI领域的"操作系统"级供应商。

**第二，Cosmos Coalition 生态联盟。** NVIDIA 同时宣布成立 **Cosmos Coalition**，成员包括 Agile Robots、Black Forest Labs、Generalist、LTX、Runway、Skild AI 等物理AI领域的关键玩家。联盟成员可以使用 Cosmos 3 技术和 NVIDIA DGX Cloud 基础设施进行大规模训练，并贡献自己的模型和研究。这种"开源+联盟"的模式，NVIDIA 曾在 CUDA 生态上成功实践，如今正在物理AI领域复制。

**第三，数据飞轮闭环。** 更多开发者使用 Cosmos 3，就会有更多行为数据流回 NVIDIA 的生态体系。这些数据对于训练下一版本的世界模型至关重要——而下一版本的 Cosmos，又会进一步巩固 NVIDIA 在物理AI基础设施层的主导地位。

## 五、影响与冲击：对机器人、自动驾驶、具身智能的意义

### 机器人行业：从"手工调参"到"数据驱动"

过去机器人控制系统的开发高度依赖工程师对特定任务的手工调参和仿真验证，泛化能力极差——一个在仓库分拣场景中表现优秀的机器人，换到另一个仓库可能完全无法工作。Cosmos 3 提供的世界模型预训练+任务微调范式，有望让机器人获得**跨场景泛化能力**，将行业从"手工调参"时代真正推向"数据驱动"时代。

### 自动驾驶：仿真数据生成的"量级飞跃"

自动驾驶训练最大的成本之一是**长尾场景数据**——极端天气、交通事故、突发障碍等罕见场景在真实道路上很难采集。Cosmos 3 的视频预测和文本生成视频能力，可以让自动驾驶公司在虚拟环境中**按需生成**这类数据，大幅压缩训练成本。

### 具身智能：降低"最后一公里"成本

具身智能（Embodied AI）的核心难题是**物理世界数据严重不足**——互联网上海量的文本和图像数据为语言和视觉模型提供了充足的训练燃料，但机器人动作数据却极度稀缺。Cosmos 3 的 omnimodel 架构通过将动作预测纳入预训练阶段，**从源头缓解了具身智能的数据饥渴问题**。

## 六、挑战与局限：不可回避的问题

尽管 Cosmos 3 带来了令人兴奋的突破，我们也需要冷静看待它面临的挑战：

**第一，动作输出的精度与实时性。** Cosmos 3 输出的动作信号最终仍需要经过控制系统的精调才能真正驱动机器人。在高精度工业场景（如精密装配手术）中，当前的动作输出精度是否足够，仍有待验证。

**第二，Sim-to-Real 迁移问题。** 即使在虚拟环境中表现优秀，模型学到的物理规律在迁移到真实世界时仍可能遇到"现实鸿沟"——虚拟物理规则和真实物理规则之间的差异，可能导致预测行为偏差。

**第三，开源协议的商用限制。** OpenMDW-1.1 许可证虽然允许商业使用，但仍有部分限制条款，企业在商用落地时需要仔细评估法律风险。

**第四，竞争者的追赶。** Google DeepMind 的 RT 系列、OpenAI 的机器人研究、Figure AI 和 1X Technologies 的具身智能团队，都在世界模型领域持续投入。Cosmos 3 的先发优势能否持续，取决于 NVIDIA 在后续迭代中的速度。

## 结语

NVIDIA Cosmos 3 的发布，是2026年物理AI领域最具标志性意义的技术事件。它不仅仅是一款更强大的模型，而是重新定义了物理AI训练的底层范式——从"在真实物理世界中慢慢积累数据"，转向"在虚拟世界模型中高效生成多样化数据，再在真实世界中微调和验证"。

当世界模型、合成数据、稀疏激活推理和开源生态这四股力量汇聚在一起，物理AI的Scaling Law或许正在像2022-2023年语言模型的Scaling Law一样，即将迎来它的爆发时刻。

黄仁勋在发布时说的那句话值得记住：**"Cosmos 3 让开发者构建感知、推理、规划和行动于物理世界的机器人，变成了从个月到天的跨越。"** 这句话背后的含义是：物理AI的iPhone时刻，或许已经到来。
